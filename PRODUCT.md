# Product

## Register

product

## Users

Software teams — product managers, developers, and technical writers — who manage product documentation, changelogs, and release notes. They work in focused sessions, often switching between editing docs, reviewing versions, and publishing releases. They value clarity, speed, and consistency.

## Product Purpose

Orbit Docs is a centralized documentation and changelog management tool. It replaces scattered markdown files and manual release processes with a structured, version-centric workflow: write docs, edit changelogs tied to app versions, and publish releases from a single source of truth.

## Brand Personality

- **Functional**: Every element serves a purpose; no decorative noise.
- **Structured**: Clear hierarchy, predictable patterns, content-first layout.
- **Refined**: Polished where it matters (interactions, readability), minimal elsewhere.

## Anti-references

- Marketing-heavy landing pages with hero metrics and gradient text.
- Cluttered dashboards with nested cards and excessive chrome.
- Generic SaaS cream-and-blue aesthetics ("AI slop" templates).
- Over-animated interfaces that slow down workflow tasks.

## Design Principles

1. **Content is the interface**: Minimize chrome; docs and changelogs should dominate the viewport.
2. **Version-centric truth**: Changelogs and releases derive from version data, not standalone documents.
3. **Consistent affordances**: Same patterns for selection, creation, and editing across all surfaces.
4. **Keyboard-first efficiency**: All primary actions reachable without leaving the keyboard.
5. **Restrained color**: Tinted neutrals with a single warm accent used sparingly for primary actions.

## Accessibility & Inclusion

- WCAG 2.1 AA compliance for all interactive elements.
- Keyboard navigation for dropdowns, modals, and tables.
- Respects `prefers-reduced-motion`.
- Sufficient color contrast on all text and focus indicators.
